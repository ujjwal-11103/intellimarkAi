Executive Summary

Following the security assessment of intellimark.ai by the Parafox / Proaxis Solutions team, a series of system hardening measures were implemented. The VAPT report originally identified 7 key vulnerabilities comprising missing security headers, CORS wildcard configurations, and subresource integrity exclusions.

Our engineering team has updated the code repository, integrated custom security headers via vercel.json, and secured external assets. As of this report, all 7 findings are 100% resolved and validated both locally and on the live production site.

Detailed Resolution Pages

V-01
Content Security Policy (CSP) Header
RESOLVED
Vulnerability Description
The application did not implement the Content-Security-Policy header, allowing unrestricted execution of cross-site scripts (XSS) and loading of unauthorized resources.
Resolution Implementation
A comprehensive Content Security Policy has been configured in vercel.json. Scripts are restricted to 'self', Google Tag Manager, and Vercel. Images are restricted to 'self', Pexels, Wikimedia, and trusted logo CDNs.
Verification Proof — Production Header
Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval' https://www.googletagmanager.com https://va.vercel-scripts.com; style-src 'self' 'unsafe-inline'; img-src 'self' data: https://www.googletagmanager.com https://*.google-analytics.com https://images.pexels.com https://upload.wikimedia.org https://vectorseek.com https://cdn.worldvectorlogo.com; connect-src 'self' https://vitals.vercel-insights.com https://www.google-analytics.com https://*.google-analytics.com; frame-ancestors 'none';
V-02
Cross-Domain CORS Misconfiguration
RESOLVED
Vulnerability Description
The application served Access-Control-Allow-Origin: * (wildcard), allowing any external domain to read cross-origin responses, which could expose private data.
Resolution Implementation
Overrode wildcard headers and explicitly configured CORS origin to allow only https://www.intellimark.ai (no trailing slash, per W3C CORS spec).
Verification Proof — Production Header
Access-Control-Allow-Origin: https://www.intellimark.ai
V-03
Anti-Clickjacking Headers
RESOLVED
Vulnerability Description
The site did not send X-Frame-Options or CSP frame-ancestors directive, allowing external domains to embed the website inside malicious iframes for UI redressing attacks.
Resolution Implementation
Set X-Frame-Options: DENY and configured frame-ancestors 'none' in the Content Security Policy to completely prevent iframe embedding.
Verification Proof — Production Headers
X-Frame-Options: DENY Content-Security-Policy: ... frame-ancestors 'none';
V-04 & V-05
Subresource Integrity (SRI) & Cross-Domain JS Inclusion
RESOLVED
Vulnerability Description
Third-party JavaScript files (Google Tag Manager) were loaded without integrity hashes, and external JS was included cross-domain without verification.
Resolution Implementation
Added crossorigin="anonymous" to GTM's script tag in index.html. Since GTM is dynamic and compiled by Google, static SRI hashes would break the script. Security is maintained via strict CSP script-src/connect-src directives limiting JS execution to trusted domains only.
Code Change — index.html
<script async src="https://www.googletagmanager.com/gtag/js?id=G-2V02R4ZW3R" crossorigin="anonymous"></script>
V-06 & V-07
HSTS & MIME-Type Sniffing Headers
RESOLVED
Vulnerability Description
HSTS (Strict-Transport-Security) was reported missing in automated scans (marked False Positive on manual verification). X-Content-Type-Options was not present, allowing potential MIME-type confusion attacks.
Resolution Implementation
Explicitly configured both Strict-Transport-Security (with 2-year max-age, includeSubDomains, preload) and X-Content-Type-Options: nosniff in vercel.json for all routes.
Verification Proof — Production Headers
Strict-Transport-Security: max-age=63072000; includeSubDomains; preload X-Content-Type-Options: nosniff
